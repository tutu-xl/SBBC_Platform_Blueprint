# 开发流程与预算

## 1. 总体交付方式

SBBC 应作为一个分阶段产品项目推进，而不是一次性整体开发。

建议路径：

- V0：先完成规划
- V1：先上线品牌与转化价值
- V2：建立运营底座
- V3：开放教师和学生的使用入口
- V4：扩展到在线学习

## 2. 建议开发流程

### 阶段 0：需求与方案

交付物：

- 产品边界确认
- PRD
- Sitemap 和模块图
- 角色与权限模型
- 数据模型草案
- 技术架构草案
- 成本估算
- 路线图

### 阶段 1：UX 与设计

交付物：

- 品牌视觉方向
- 页面级线框图
- 核心后台线框图
- Design System 基础规则
- 响应式布局规范

### 阶段 2：工程搭建

交付物：

- 仓库结构和 CI/CD
- 环境配置
- 数据库 schema
- Auth 和 RBAC 基础能力
- 共享 UI 基础组件

### 阶段 3：按版本实施

交付物：

- V1 官网与转化
- V2 管理后台与 CRM
- V3 教师端与学生 Portal
- V4 在线课程

### 阶段 4：测试与上线

交付物：

- 测试计划
- UAT
- 正式上线
- 上线后监控与修复

## 3. 各版本工作量预估

以下是规划级预估，不是最终固定报价。

### V0

- 需求、架构、路线图、预算规划
- 预估时间：`1 到 2 周`

### V1

- 官网、课程页、文章系统、表单、线索收集
- 预估时间：`3 到 6 周`

### V2

- 管理后台、CRM、学生档案、课时管理
- 预估时间：`4 到 8 周`

### V3

- 教师工作台、学生 Portal、财务基础、证书
- 预估时间：`4 到 8 周`

### V4

- 录播课程、直播接入、预约自动化
- 预估时间：`6 到 12 周`

## 4. 团队配置建议

### 精简团队

- 1 名产品 / 技术负责人
- 1 名全栈工程师
- 1 名兼职设计师

适合：

- 节奏相对稳，预算控制更强

### 平衡团队

- 1 名产品负责人
- 1 名设计师
- 2 名工程师
- 兼职 QA

适合：

- 多模块并行推进
- 更快完成版本交付

## 5. 开发预算模型

以下为澳元 AUD 的粗略规划级预估。

### 方案 A：精简阶段式开发

- V0 规划：`AUD 2,000 到 5,000`
- V1 开发：`AUD 8,000 到 18,000`
- V2 开发：`AUD 12,000 到 25,000`
- V3 开发：`AUD 12,000 到 25,000`
- V4 开发：`AUD 18,000 到 40,000`

如果全部阶段都完成，总额大约：

- `AUD 52,000 到 113,000`

### 方案 B：更强生产级交付

- V0 规划与架构：`AUD 4,000 到 8,000`
- V1：`AUD 15,000 到 30,000`
- V2：`AUD 20,000 到 40,000`
- V3：`AUD 20,000 到 40,000`
- V4：`AUD 30,000 到 60,000`

总额大约：

- `AUD 89,000 到 178,000`

## 6. 预算策略建议

- 把 V0 视为必要的规划投入
- 不要在 V1 之前就一次性为 V3/V4 的复杂度买单
- 以版本里程碑签约和付款
- 每完成一个版本后重新估算下一阶段

## 7. 影响成本的主要风险

- 产品方向频繁变动
- 财务需求后期突然变复杂
- 自定义支付流程过多
- LMS 能力要求过深
- 多语言内容量大且维护复杂
- 现有网站代码若质量不稳定，迁移和接手成本会增加
- 过早自建直播和视频能力
