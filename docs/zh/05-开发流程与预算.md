# 开发流程与预算

## 1. 总体交付方式

SBBC 应作为一个分阶段产品项目推进，而不是一次性整体开发。

建议路径：

- V0：先完成规划
- V1：先上线品牌与转化价值
- V2：建立运营底座
- V3：开放教师和学生的使用入口
- V4：扩展到在线学习

## 2. 建议开发流程

### 阶段 0：需求与方案

交付物：

- 产品边界确认
- PRD
- Sitemap 和模块图
- 角色与权限模型
- 数据模型草案
- 技术架构草案
- 成本估算
- 路线图

### 阶段 1：UX 与设计

交付物：

- 品牌视觉方向
- 页面级线框图
- 核心后台线框图
- Design System 基础规则
- 响应式布局规范

### 阶段 2：工程搭建

交付物：

- 仓库结构和 CI/CD
- 环境配置
- 数据库 schema
- Auth 和 RBAC 基础能力
- 共享 UI 基础组件

### 阶段 3：按版本实施

交付物：

- V1 官网与转化
- V2 管理后台与 CRM
- V3 教师端与学生 Portal
- V4 在线课程

### 阶段 4：测试与上线

交付物：

- 测试计划
- UAT
- 正式上线
- 上线后监控与修复

## 3. 各版本工作量预估

以下是规划级预估，不是最终固定报价。

与前一版粗略估算相比，这里我有意把开发时间整体拉长到原来的 2 倍到 3 倍左右。原因是实际项目中一定会包含反复沟通、设计调整、权限细化、测试、上线准备、数据修正和验收，这些通常会显著拉长周期。

### V0

- 需求、架构、路线图、预算规划
- 预估时间：`2 到 4 周`

### V1

- 现有官网完善、支付、基础管理后台、学生 Portal、教师端
- 预估时间：`6 到 15 周`

### V2

- CRM、客服工作流、学生档案、套餐与课时、排课支持、教务运营
- 预估时间：`8 到 20 周`

### V3

- 各部门 Dashboard、财务基础、审批流程、分析报表、证书
- 预估时间：`8 到 20 周`

### V4

- 录播课程、进度追踪、在线学习扩展
- 预估时间：`12 到 30 周`

## 4. 团队配置建议

### 精简团队

- 1 名产品 / 技术负责人
- 1 名全栈工程师
- 1 名兼职设计师

适合：

- 节奏相对稳，预算控制更强

### 平衡团队

- 1 名产品负责人
- 1 名设计师
- 2 名工程师
- 兼职 QA

适合：

- 多模块并行推进
- 更快完成版本交付

## 5.1 这些价格是怎么估出来的

以上区间不是拍脑袋写的，而是结合当前澳洲公开市场参考，再换算成“真实项目交付价格”。

公开参考信号包括：

- SEEK 显示澳洲 Software Engineer 平均年薪大约在 `AUD 105,000 到 125,000`
- SEEK 显示澳洲 Web Developer 平均年薪大约在 `AUD 80,000 到 100,000`
- PayScale 显示澳洲 Software Developer 平均年薪大约在 `AUD 79,830`

为什么项目价格会高于“把工资按月折算”：

- 项目里不只是编码
- 还包含需求沟通、设计、修改、测试、上线、项目协调、风险缓冲
- 如果按外包、顾问或定制开发模式报价，通常会显著高于纯工资成本

所以这里的“市场参考价”更接近真实定制开发市场区间；而“7 折左右的价格”则更适合作为关系合作价、创业友好价或前期阶段合作价。

## 6. 预算策略建议

- 把 V0 视为必要的规划投入
- 不要在 V1 之前就一次性为 V3/V4 的复杂度买单
- 以版本里程碑签约和付款
- 每完成一个版本后重新估算下一阶段

## 7. 影响成本的主要风险

- 产品方向频繁变动
- 财务需求后期突然变复杂
- 自定义支付流程过多
- LMS 能力要求过深
- 多语言内容量大且维护复杂
- 现有网站代码若质量不稳定，迁移和接手成本会增加
- 过早自建直播和视频能力
