# SBBC 产品需求文档

本文件定义“做什么”。关于“什么时候做”，请看 [02-路线图-V0-V4.md](./02-路线图-V0-V4.md)。关于“怎么做”，请看 [03-架构与门户策略.md](./03-架构与门户策略.md)。

## 1. 产品定义

SBBC 从一开始就不应只被定义为一个官网，而应被规划为一套逐步扩展的平台型软件。

平台范围可以分为以下几类：

**对外展示与获客**

- 品牌官网
- 内容系统与搜索引擎优化（SEO）
- 咨询与试听转化链路
- 周边展示入口；购买跳转到独立的 `SBBC Store`

**招生与报名转化**

- CRM 意向客户记录管理
- 试听预约与跟进
- 学生报名与支付

**教学与日常运营**

- 管理后台
- 教务运营
- 教师工作台
- 学生端

**数据、财务与管理分析**

- 财务记录与报表
- BI 分析与管理看板

**后续学习能力扩展**

- 录播学习能力
- 直播学习能力

## 2. 产品目标

### 2.1 业务目标

- 建立高端、可信、双语友好的品牌形象
- 提升咨询到报名的转化效率
- 规范学生、课程、课时和排课管理
- 减少人工跟进、人工统计、人工沟通带来的负担
- 为后续线上课程与混合式教学建立可扩展基础

### 2.2 产品目标

- 公共网站要同时承担品牌展示、信任建立、招生转化三类职责
- 内部系统要支持真实日常运营，而不是只做展示型后台
- 学生在一个入口内完成学习、查看课时、查看付款和进入课程
- 教师在系统中高效完成授课相关事务
- 从 V0 开始为后面的 LMS、财务、自动化预留架构空间

## 3. 核心用户角色

- Visitor：普通访客，浏览网站和内容
- Student：报名课程的学员，可登录学生端查看课程、课时、课表、反馈、付款记录等信息
- Parent：未成年学生的家长或监护人
- Teacher：只能看到自己相关学生和课程的教师
- Teaching Assistant：助教或运营支持角色，可负责分配给自己的咨询线索跟进
- Admin：负责学生、课程、线索、内容和日常运营管理的管理员
- Super Admin：最高权限角色，管理权限、配置、财务规则和审计

注：`Lead` 指 CRM 中的意向客户记录 / 咨询线索，不是登录角色。

## 4. 核心业务域

### 4.1 公共网站与品牌

- 首页
- 课程体系页
- 课程详情页
- Learning Hub / 文章中心
- 关于 SBBC
- 教师团队
- 咨询预约
- 试听申请
- 联系方式 / 微信 / 社交入口
- 周边商品展示页
- `SBBC Store` 跳转入口
- 主站只做展示和导流，不承接商品下单、库存和履约

### 4.2 CRM 与招生转化

- 意向客户记录 / 咨询线索列表
- 来源追踪
- 咨询记录
- 试听预约流程
- 跟进状态流转
- 报名转化统计
- 助教 / 运营人员分配与跟进

### 4.3 学生管理

- 学员档案
- 联系方式与监护人信息
- 课程与套餐信息
- 课时消耗
- 出勤记录
- 作业与反馈
- 考试准备和成绩
- 证书

### 4.4 课时与排课

- 教师课表
- 学生预约
- 试听时段管理
- 已预约 / 已完成 / 请假 / 取消 / 补课等状态
- 扣课、补回、调整

### 4.5 教师工作台

- 我的学生
- 今日课程
- 提交课后反馈
- 出勤登记
- 作业与下节课计划

### 4.6 学生 Portal

- 我的课程
- 剩余课时
- 我的课表
- 上课历史
- 教师反馈
- 作业
- 账单与付款记录
- 预录制课程
- 直播课程入口
- 积分余额
- 积分获得记录

### 4.7 内容系统

- 文章编辑
- SEO 字段
- 页面网址（Slug）管理
- 封面图
- 推荐课程
- 阅读统计

### 4.8 财务

- 支付记录
- 报名订单
- 发票 / 收据
- 退款 / 调整
- 收入看板
- 应收余额

### 4.9 在线学习

- 预录制课程
- 视频进度
- 直播课入口
- 录播回放
- 课程完成状态

### 4.10 积分与周边展示

- 由于学校主体通常不适合直接做商品销售，`SBBC Store` 建议作为独立网站或独立公司来承接周边销售
- SBBC 主站只做周边商品展示和跳转入口，不直接处理周边商品的下单、库存、发货和售后
- 用户点击周边入口后，跳转到独立的 `SBBC Store` 完成购买
- 学生积分账户
- 积分获得规则和积分记录
- 积分来源可以包括报名课程、课堂表现、比赛成绩等，不同来源可以对应不同积分规则
- 积分先记录在学生端，兑换功能后续版本再做
- 后续再设计积分与 `SBBC Store` 的衔接方式，例如优惠券、兑换码或其他兑换流程

## 5. 功能需求

### 5.1 公共网站

- 品牌气质要体现高端、专业、可信
- 兼顾中英文展示
- 在移动端有良好体验
- 课程内容结构清晰
- 支持课程标签与筛选
- 咨询和试听入口明显
- 支持 SEO 友好的内容结构

### 5.2 CRM

- 意向客户来源至少包括：官网、微信、小红书、公众号、手动录入
- 意向客户状态至少包括：新咨询、已联系、已约试听、已试听、已报名、未成交
- 支持备注、负责人、跟进历史
- 支持将意向客户记录分配给助教或运营人员跟进
- 可以看到最近互动和下一步建议

### 5.3 学生档案

- 中文名
- 英文名
- 邮箱
- 电话
- 微信
- 家长联系方式
- 当前课程
- 剩余课时
- 开始与结束日期
- 授课老师 / 助教
- 上课记录
- 反馈记录
- 作业记录
- HSK / BCT / 模考记录

### 5.4 课时与课程管理

- 日历式课程记录
- 课程状态管理
- 试听预约
- 教师空闲时段
- 学生预约申请表
- 管理员、老师、学生的邮件通知

### 5.5 教师端

- 仅看到自己负责的学生和课程
- 今日课程列表
- 课后反馈表单
- 出勤登记
- 作业与下节课计划

### 5.6 学生 Portal

- 安全登录
- 查看课程与报名状态
- 查看剩余课时
- 查看历史反馈
- 查看付款与发票
- 下载证书
- 观看录播
- 进入直播课
- 查看积分余额
- 查看积分获得记录

### 5.7 管理后台 Dashboard

- 查看本月新增学员、活跃学员和在读学员
- 查看最新咨询、试听预约和报名转化情况
- 查看课程、课时、排课和教师工作概况
- 查看基础收入、订单和付款记录
- 查看内容、网站访问和活动数据概览
- 后续可扩展为更完整的 BI 看板，具体版本安排见路线图

### 5.8 课程管理

- 课程名称
- 标签：HSK、Business Chinese、VIP、Executive、Children
- 封面图
- 价格
- 课时数
- 推荐课程开关
- 课程说明
- 授课模式：线下、线上直播、录播、混合

### 5.9 内容管理

- 搜索结果标题（Meta Title）
- 搜索结果描述（Meta Description）
- 页面网址（Slug）
- 作者
- 分类
- 阅读量
- 图片说明文字（Alt 文本）
- 结构化数据标记（Schema.org）
- 中英文页面对应关系

### 5.10 证书生成

- 根据课程完成状态生成结课证书
- 包含学生姓名、课程名称、完成日期、SBBC Logo

## 6. 非功能性要求

- 角色权限控制
- 敏感操作审计日志
- 安全认证和找回密码
- 数据备份与恢复
- 移动端性能良好
- 后台对中文运营人员友好
- 架构支持后续 LMS 和财务扩展
