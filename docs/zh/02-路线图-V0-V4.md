# SBBC 路线图：V0 到 V4

## 总体原则

SBBC 不能按“一次性全部开发完成”的思路推进，而应按版本逐步交付。

另外，当前官网已经基本成型，并不是从零开始，所以后续版本应该建立在“现有网站可继续沿用和优化”的前提上。

核心原则：

- 先把方向和架构想清楚
- 先做最能产生业务价值的部分
- 每一版都为下一版铺路

## V0：规划与基础设计

目标：

- 明确平台边界、版本路线、技术方案和预算模型

内容包括：

- 产品需求梳理
- 信息架构
- 主要业务流程
- 核心数据模型草案
- 技术栈决策
- 服务与成本估算
- 开发计划

输出物：

- 项目规划文档库
- 需求文档
- 初步架构文档
- 分阶段实施方案

## V1：现有网站完善 + 支付 + 基础后台与 Portal

目标：

- 以现有官网为基础，做出第一个真正“能用起来”的版本

内容包括：

- 保留并优化现有官网
- 继续完善咨询与试听申请流程
- 增加报名支付能力
- 支持当前主要授课模式：一对一和小班直播授课
- 直播课前期通过 Zoom、腾讯会议等第三方链接接入
- 基础管理后台
- 基础学生 Portal
- 基础家长 / 监护人付款与确认流程
- 基础教师端
- 基础助教 / 运营支持入口
- 学生资料与报名记录
- 课时 / 套餐管理
- 老师与学生的基础关联
- 助教与学生 / 咨询记录的基础关联
- 基础订单与付款记录

必须达成的结果：

- 官网继续可用并保持品牌感
- 成年学生可以自行报名、付款，并管理订单和发票
- 未成年学生可以发起报名 / 选择课程，但付款、确认和同意条款由家长或监护人完成
- 管理员可以管理核心数据
- 老师、学生、家长 / 监护人和助教开始进入自己的工作区或基础流程
- 助教可以参与基础咨询跟进、学生信息查看和运营支持

## V1 明确不包含

以下内容不放在 V1：

- 录播课程库
- 平台自建的直播视频基础设施
- 在 SBBC 主平台内处理 `SBBC Store` 的下单、库存、履约和售后
- 主站周边展示与 `SBBC Store` 跳转链路
- 学生积分账户与积分获得记录
- 积分兑换或周边兑换流程
- 完整家长 Portal
- 高级财务报表
- 更复杂的自动化和审批流
- 原生移动 App
- 多校区或多品牌支持
- 复杂的分销或佣金体系

## V2：CRM + 教务运营 + 排课支持

目标：

- 把招生跟进、教务协调、课时处理和排课支持拉进系统

内容包括：

- 咨询客户 / 意向客户管理
- 客服 / 课程顾问工作流
- 学生档案
- 老师档案
- 助教协作工作流
- 课程与套餐管理
- Lesson 记录
- 课时扣减 / 补回
- 排课支持
- 请假 / 补课 / 取消处理
- 跟进记录与备注
- 基础通知能力
- 学生积分账户
- 积分获得规则与积分记录
- 积分来源可以包括报名课程、课堂表现、比赛成绩等，不同来源可以对应不同积分规则
- 积分在学生端可见，但暂不做兑换

必须达成的结果：

- 学生、老师、助教、咨询客户的数据集中管理
- 排课与教务协作更顺畅
- 人工沟通和手工记录明显减少
- 学生可以看到课程和学习相关行为获得的积分

## V3：深度 BI 仪表盘 + 财务基础 + 审批流程

目标：

- 将业务运营数据转化为战略洞察，通过全方位 BI 看板支持决策

内容包括：

- 核心团队 BI 大盘：涵盖学生、教师、营销、活动和线上学习的深度分析
- 营销来源追踪与归因（Google, Facebook, 小红书等）
- 网站表现与转化漏斗分析
- 角色化看板（教务、财务、创始人各取所需）
- 文化活动 ROI 追踪
- 费用报销与 OA 审批流
- 发票 / 收据管理
- 自动证书生成
- 主站周边商品展示与 `SBBC Store` 跳转设计
- 明确周边展示入口、链接位置和跳转规则
- 评估后续是否需要优惠券 / 兑换码与独立 `SBBC Store` 衔接
- 使用现代可视化库（如 Tremor）打磨报表 UI

必须达成的结果：

- 领导层拥有清晰的数据洞察（LTV, CAC, 留存/流失分析）
- 营销和活动的投资回报率 (ROI) 可量化
- 审批和财务流程线上化，透明可追溯
- SBBC 主站可以展示周边，但购买行为保留在独立的 `SBBC Store` 网站

## V4：在线课程扩展

目标：

- 在业务成熟后，再把平台扩展到录播课程和更完整的在线学习能力

内容包括：

- 录播课程库
- 视频权限控制
- 学习进度追踪
- 完成状态记录
- 后续可增加阶段检测或测验
- 直播仍可优先使用 Zoom、腾讯会议等第三方链接

必须达成的结果：

- SBBC 在准备好后可以推出系统化录播课程
- 平台从教务系统自然延伸到在线学习

## V5 以后可能扩展

- 高级财务报表
- Referral / 推荐奖励体系
- 高级数据分析
- AI 学习辅助
- 多校区 / 多品牌管理
- 原生移动应用
- 如有需要，再做更深的 `SBBC Store` 对接，例如积分兑换规则或优惠券跳转

## 版本顺序建议

- V1 重点不是重做官网，而是把当前业务流程跑通
- V1 暂时不需要录播课程
- V1 先支持当前主要模式：一对一和小班直播授课
- V2 要把客服、助教、排课、教务协作的模型打牢
- V2 可以加入学生积分账户和积分获得记录，但暂时不做兑换
- V3 可以设计主站周边展示，以及跳转到独立 `SBBC Store` 的规则
- SBBC 主平台不直接处理周边商品的下单、库存、发货和售后
- V3 之前必须先稳定 RBAC、学生、课程、课时、支付模型
- V4 不宜在权限、支付、内容模型没稳定前过早启动
