# SBBC 路线图：V0 到 V4

## 总体原则

SBBC 不能按“一次性全部开发完成”的思路推进，而应按版本逐步交付。

核心原则：

- 先把方向和架构想清楚
- 先做最能产生业务价值的部分
- 每一版都为下一版铺路

## V0：规划与基础设计

目标：

- 明确平台边界、版本路线、技术方案和预算模型

内容包括：

- 产品需求梳理
- 信息架构
- 主要业务流程
- 核心数据模型草案
- 技术栈决策
- 服务与成本估算
- 开发计划

输出物：

- 规划仓库
- 需求文档
- 初步架构文档
- 分阶段实施方案

## V1：高端官网 + 招生转化 + 基础 CMS

目标：

- 先把品牌、课程展示和获客入口做好

内容包括：

- 新首页
- About / Team / Programs / Contact 页面
- 课程详情页
- Learning Hub 文章系统
- 文章和课程 SEO 字段
- 咨询表单
- 试听申请表单
- Leads 收集到 CRM
- 基础内容管理后台

必须达成的结果：

- 品牌质感显著提升
- 转化入口顺畅
- 内容结构对 SEO 友好

## V2：管理后台 + CRM + 学生档案 + 课时管理

目标：

- 把日常运营从零散手工状态拉进系统

内容包括：

- Admin Dashboard
- Leads 管理
- 学生档案
- 课程与套餐管理
- 报名 / 订单记录
- Lesson 记录
- 课时管理
- 教师档案
- 跟进记录与备注
- 基础邮件通知

必须达成的结果：

- 学生和线索管理集中化
- 管理员工作负担下降
- 后台数据开始可追踪

## V3：教师工作台 + 学生 Portal + 财务基础

目标：

- 不只是管理员使用系统，教师和学生也开始进入平台

内容包括：

- 教师工作台
- 教师课表与反馈
- 出勤登记
- 学生 Portal
- 课程与课时查看
- 账单与付款记录
- 发票 / 收据展示
- 证书生成

必须达成的结果：

- 教师可以在系统内完成核心教学事务
- 学生有更好的服务体验
- 平台从后台工具升级为面向用户的软件产品

## V4：在线课程与排课自动化

目标：

- 支持线上化和规模化教学能力

内容包括：

- 教师可用时段管理
- 学生自助预约
- 录播课程库
- 视频权限控制
- 直播课接入
- 学习完成状态
- 课程进度记录
- 回放入口

必须达成的结果：

- 线上课程收入模型可以真正落地
- 平台具备从教务系统向 LMS 延伸的能力

## V5 以后可能扩展

- 家长 Portal
- 高级财务报表
- Referral / 推荐奖励体系
- AI 学习辅助
- 多校区 / 多品牌管理
- 原生移动应用

## 版本顺序建议

- V1 不要等 V2 全部想清楚才开始
- V2 要把后续 Portal 需要的数据模型打牢
- V3 之前必须先稳定 RBAC、学生、课程、课时模型
- V4 不宜在权限、支付、内容模型没稳定前过早启动
