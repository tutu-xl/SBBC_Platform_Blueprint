# SBBC 路线图：V0 到 V4

## 总体原则

SBBC 不能按“一次性全部开发完成”的思路推进，而应按版本逐步交付。

另外，当前官网已经基本成型，并不是从零开始，所以后续版本应该建立在“现有网站可继续沿用和优化”的前提上。

核心原则：

- 先把方向和架构想清楚
- 先做最能产生业务价值的部分
- 每一版都为下一版铺路

## V0：规划与基础设计

目标：

- 明确平台边界、版本路线、技术方案和预算模型

内容包括：

- 产品需求梳理
- 信息架构
- 主要业务流程
- 核心数据模型草案
- 技术栈决策
- 服务与成本估算
- 开发计划

输出物：

- 规划仓库
- 需求文档
- 初步架构文档
- 分阶段实施方案

## V1：现有网站完善 + 支付 + 基础后台与 Portal

目标：

- 以现有官网为基础，做出第一个真正“能用起来”的版本

内容包括：

- 保留并优化现有官网
- 继续完善咨询与试听申请流程
- 增加报名支付能力
- 基础管理后台
- 基础学生 Portal
- 基础教师端
- 学生资料与报名记录
- 课时 / 套餐管理
- 老师与学生的基础关联
- 基础订单与付款记录

必须达成的结果：

- 官网继续可用并保持品牌感
- 学生可以报名并付款
- 管理员可以管理核心数据
- 老师和学生开始进入自己的 Portal

## V2：CRM + 教务运营 + 排课支持

目标：

- 把招生跟进、教务协调、课时处理和排课支持拉进系统

内容包括：

- 咨询客户 / 意向客户管理
- 客服 / 课程顾问工作流
- 学生档案
- 老师档案
- 助教协作工作流
- 课程与套餐管理
- Lesson 记录
- 课时扣减 / 补回
- 排课支持
- 请假 / 补课 / 取消处理
- 跟进记录与备注
- 基础通知能力

必须达成的结果：

- 学生、老师、助教、咨询客户的数据集中管理
- 排课与教务协作更顺畅
- 人工沟通和手工记录明显减少

## V3：各部门 Dashboard + 财务基础 + 审批流程

目标：

- 从日常运营工具升级为可支撑部门管理的内部系统

内容包括：

- 不同角色的 Dashboard
- 部门负责人 Dashboard
- 教务负责人 Dashboard
- 财务 Dashboard
- 创始团队 / 管理层 Dashboard
- 报销与基础审批流程
- 发票 / 收据管理
- 证书生成
- 更完整的数据分析与图表

必须达成的结果：

- 不同角色能看到不同后台首页和数据视图
- 财务与审批能力开始在系统内形成闭环
- 管理层能更清晰地看运营状态

## V4：在线课程扩展

目标：

- 在业务成熟后，再把平台扩展到录播课程和更完整的在线学习能力

内容包括：

- 录播课程库
- 视频权限控制
- 学习进度追踪
- 完成状态记录
- 后续可增加阶段检测或测验
- 直播仍可优先使用 Zoom、腾讯会议等第三方链接

必须达成的结果：

- SBBC 在准备好后可以推出系统化录播课程
- 平台从教务系统自然延伸到在线学习

## V5 以后可能扩展

- 家长 Portal
- 高级财务报表
- Referral / 推荐奖励体系
- AI 学习辅助
- 多校区 / 多品牌管理
- 原生移动应用

## 版本顺序建议

- V1 重点不是重做官网，而是把当前业务流程跑通
- V1 暂时不需要录播课程
- V1 先支持当前主要模式：一对一和小班直播授课
- V2 要把客服、助教、排课、教务协作的模型打牢
- V3 之前必须先稳定 RBAC、学生、课程、课时、支付模型
- V4 不宜在权限、支付、内容模型没稳定前过早启动
